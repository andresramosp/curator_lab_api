note this error:   {"tagName": "fountain", "isSubtype": true},. Fountain is just a component or part of a park, not a sub type of park. I will remind you the rules here, then you can continue, with the semantic field "cultural / religious"

### Rules for Semantic Relationships:
1. **Subtype Definition:**
   - A tag is a subtype if it is ontologically contained in the term (e.g., "cat" is a subtype of "feline").
   - A tag is more specific than the term (e.g., "white cat" is a subtype of "cat").
   - When a tag is a compound term (e.g., "man with diamond"), focus on the relevant part. If the relevant part (e.g., "diamond") is a subtype of the term (e.g., "mineral"), the compound term is also a subtype.

2. **Synonyms:**
   - Exact or near-synonyms (e.g., "sea" and "ocean") should be treated as subtypes, provided they do not overgeneralize.

3. **Non-Subtypes:**
   - A tag is not a subtype if it represents a part of the term rather than the term itself (e.g., "leg" is not a subtype of "person").
   - A tag is not a subtype if it is from a different semantic domain (e.g., "Washington" is not a subtype of "USA").

4. **Ambiguity:**
   - If a tag could belong to multiple terms, choose the most relevant one based on its semantic relationship.

5. **Action Tags:**
   - Tags describing actions (e.g., "child playing") can have subtypes if they specify the type of action (e.g., "child playing soccer" is a subtype of "child playing").
   - Tags describing different actions (e.g., "child studying") are not subtypes of the original action tag.

6. **Contraejemplos:**
   - A tag that describes a different type of the same term is not a subtype (e.g., "green cat" is NOT a subtype of "white cat"). Each is a separate subtype of "cat".
   - Avoid relationships that introduce logical errors, such as considering "child studying" as a subtype of "child playing".
